My Architecture
Architecture
SSA — Single Script Architecture

I split game systems into services. Each service gets their own module and initialized depending on who owns it — some run on the client, some on the server, some on both. For example a build system would have a BuildServiceClient and a BuildServiceServer, or just one of them if the service only needs to run on one side. The script that boot everything up handle this automatically there will be only one local script in starterplayer and one server on serverscriptservice that initiliazes the services.

-- server boots server-side services
BuildServiceServer:init()
-- client boots client-side services
BuildServiceClient:init()
-- some services only need one side
somecoolservice:init()  -- server only
