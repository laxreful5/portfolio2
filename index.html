How I Code

My Approach

SSA — Single Script Architecture

Instead of having random scripts thrown all over your game, SSA keeps everything in one organized place. Each feature lives in its own small module A single main script inits all the modules and runs everything. It makes the whole project way easier to manage for all scripters, easier to fix when something breaks, and way easier to add new stuff later.

OOP — Object Oriented Programming

When something in the game needs to repeat itself — like different stands, different enemies,etc — I use OOP. I write the code once, and then create as many copies of it as I need. Less repeated code, cleaner structure, and it scales well no matter how big the game gets.

My Setup

My Workflow

I write all my code in Visual Studio Code instead of the default Roblox Studio editor — it's just way better for writing code. To get my code into Roblox, I use Rojo, which syncs everything from VS Code into Studio automatically in real time, so I never have to copy-paste anything. I also use Selene which checks my code while I write it and tells me if I'm making a mistake or writing something in a bad way, finally StyLua handles all the formatting — it makes sure my code always looks clean and consistent without me having to think about it.
